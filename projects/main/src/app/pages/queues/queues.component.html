Day 35: Queues

Focusing on standard operations, sliding window techniques, and circular buffers.

Topic Tags & Concepts

Sliding Window

A technique used to track a subset of data in an array or string to reduce time complexity from O(N²) to O(N).

Formula: window_sum = window_sum - element_out + element_in

Double-Ended Queue (Deque)

A linear data structure where elements can be added or removed from either the front or the back.

Common Pattern: Monotonic Deque for Sliding Window Max/Min.

Queue (FIFO)

First-In-First-Out operations. Core methods: Enqueue (push) and Dequeue (pop).

Complexity: O(1) for both push and pop.

Circular Buffer

A fixed-size queue that wraps around to the beginning, optimizing space in memory-constrained environments.

Index: i = (i + 1) % capacity

Implement Queue using Stacks

Implement a first-in-first-out (FIFO) queue using only two stacks.

expand_more
lightbulb Approach & Logic
  • Use two stacks: input and output.
  • For push, simply add to the input stack.
  • For pop/peek, if output stack is empty, transfer all elements from input stack to output stack.
code Blueprint Code 
class MyQueue {
  private Stack<Integer> input = new Stack<>();
  private Stack<Integer> output = new Stack<>();

  public void push(int x) {
      input.push(x);
  }

  public int pop() {
      peek();
      return output.pop();
  }

  public int peek() {
      if (output.empty())
          while (!input.empty())
              output.push(input.pop());
      return output.peek();
  }
}
